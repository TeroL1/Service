# Changelog

Здесь хранятся все изменения, связанные с проектом.

## [0.1.0] - 20.01.2026

### Начальная версия проекта
- Поддержка API (/health, /documents, /ask, /stats, /reset)
- Локальная LLM, работающая на английском языке ("flan-t5-base")
- Поддержка только .txt формата документов

## [0.2.0] - 26.01.2026

### Added
- Добавлена поддержка .pdf и .docx

### Changed
- Обновлены зависимости "requirements.txt"
- Сервис перешел на модель "Llama-3.1-8B-Instruct" через Hugging Face API
- Основной язык сервиса - русский (Хотя данная модель поддерживает и другие языки)
- Используется новый "SentenceTransformer" - "paraphrase-multilingual-MiniLM-L12-v2"

## [0.3.0] - 28.01.2026

### Added
- Добавлен новый модуль для работы с телеграм ботом, весь базовый функционал API доступен в боте (кроме вывода ошибок, ожидания загрузки и прочего)
- Способы запуска разделены в main.py

### Fixed
- Увеличено число top_k в поиске по базе и в rerank, что позволяет чаще находить искомый контент

## [0.3.1] - 29.01.2026

### Added
- Добавлена функция для переформулировки вопроса (пока что не используется)

### Changed
- Новый rerank (прошлый не учитывал формы слов и прочее)

### Fixed
- Улучшены некоторые сообщения в чате с пользователем в bot.py

## [1.0.0] - 01.02.2026

### Added
- Добавлена изоляция: теперь каждый пользователь бота в телеграме имеет свой личный список документов
- Добавлена таблица настроек пользователя: переформулировка вопроса и top_k в search и rerank. Пока что доступ к настройкам ограничен, однако для их применения уже все есть, осталось только написать API и его обработку ботом.
- Используется новый "SentenceTransformer" - "paraphrase-multilingual-mpnet-base-v2". Новая модель значительно повысила качество на тестовых файлах (краткие конспекты в формате PDF дисциплин "Теория Вероятностей" и "Основы Функционального Анализа", которые читаются в СПБПУ Петра Великого). Модель начала видеть больше контекста, однако, качество все еще не очень высокое

### Changed
- Сервис перешел с работы на CPU с FAISS на прямое взаимодействие с PostgreSQL (поддерживающий работу с векторами)

### Fixed
- API теперь должен принимать ID пользователя
